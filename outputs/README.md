# `outputs/`

Single root for **everything machine-generated** in this project: research /
notebook artifacts and curated result artifacts. The root is relocatable with
`REFLECT_OUTPUT_ROOT`.

What is tracked in git: lightweight, human-readable results only, that is CSVs,
figures (PNG), reports (MD), and summary JSON. Large regenerable artifacts (raw
scene-graph `.jsonl`, pickles, model-call caches, videos, `.bak` backups) are
gitignored and are produced locally when you run the pipelines. Full datasets and
model checkpoints are never tracked (see
[`docs/data_and_artifacts.md`](../docs/data_and_artifacts.md)).

Two top-level areas:

- `analysis/` - artifacts produced by the research notebooks and perception
  pipelines, one sub-directory per experiment, named `<domain>_<topic>`
  (`llm_*`, `perception_*`, `vlm_*`).
- `results/` - curated CSV/JSON/MD result artifacts kept for interpretation and
  auditability.

## `analysis/`

Contents below list what is tracked; anything in parentheses is gitignored and
regenerated by the producer.

| Sub-directory | Produced by | Tracked contents |
|---|---|---|
| `llm_calibration_target_distribution/` | `sim_uncertainty_calibration_target.ipynb` (Experiment 16) | Confidence-calibration study across failure-reasoning types: per-model metric CSVs, reliability/ECE plots, `EXPERIMENT_16_REPORT.md` (LLM cache and question-set `.jsonl` gitignored) |
| `llm_calibration_standard_entropy/` | `sim_uncertainty_calibration_standard.ipynb` | Same models scored on a general MCQA benchmark: metric CSVs, plots, panel-solvability JSON (LLM cache gitignored) |
| `llm_calibration_robo2vlm/` | `bench_llm_scene_uncertainty.ipynb` | Can candidate LLMs read the scene, and is their uncertainty trustworthy: scorecard, Robo2VLM eval/panel results JSON, reliability/conformal plots |
| `llm_local_uncertainty_measurement/` | `sim_validation.ipynb`, `reflect-sim` (`local_uncertainty_root()`) | Full-dataset sim validation with multi-plan replanning: results JSON, checkpoint, episode metrics |
| `llm_sim_validation_traces/<task>/<episode>/` | `reflect-sim` sim validation pipeline (`sim_validation_trace_dir()`) | Per-task/episode LLM reasoning traces (`llm_trace.json`) |
| `perception_scene_graph_generation_cg_yoloe_track/` | reference scene-graph build (ConceptGraphs + YOLOE tracking) | Cached real-data scene-graph tables (`real_data_scene_graphs_*.csv`; zipped raw cache gitignored) |
| `perception_scene_graph_generation_eai_track/<ts>_<task>/` | `perception_yoloe_real.ipynb` / real-world perception pipeline | Per-episode run metadata and tables (`run_metadata.json`, `data`; raw `jsonl/` frame dumps gitignored), one timestamped dir per episode |
| `vlm_capability_measurement/` | `bench_vlm_captioner.ipynb` | VLM captioner comparison on YOLOE+SAM crops: `captions.json`, `judge.json` (`crops.pkl` gitignored) |

The ConceptGraphs 3D scene map (`perception_conceptgraphs_robo2vlm/scene_map.pkl.gz`)
is gitignored, so that directory only appears after you regenerate it with
`perception_conceptgraphs_real.ipynb`.

## `results/`

Curated artifacts (inventory in [`analysis/README.md`](../analysis/README.md)):

- `results/sim_data/` - per-episode pipeline outputs, human evaluations, error-taxonomy
  annotations, custom taxonomy codes, the reasoning-effort sweep, and annotation summaries.
- `results/real_world/` - final real-world results with failure explanations, human
  scores, and round-1 error-taxonomy triage.

## Notes

- Relocate the whole tree with `REFLECT_OUTPUT_ROOT`; the edge-LLM cache also
  honors `REFLECT_EDGE_CACHE`.
- All paths are resolved through `reflect.core.paths` (notebooks and CLIs emit
  exactly the names above; on-demand model-call caches land under
  `analysis/caches/`). The two `perception_scene_graph_generation_*` directories
  are hand-placed historical data with no producer in the current code base.
